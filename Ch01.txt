[1]. C언어가 사용하는 파일들 

1. source 파일
	(1). header 파일
		.h를 확장자를 가지며 다른 source파일에서 해당 함수와 변수를 사용할 수 있도록함 
	(2). implementation 파일
		.c를 확장자로 가지며 header파일에 선언된 함수와 변수를 이용하여실제 구현을 한 파일
		
2. object 파일
	.o, .obj를 확장자로 가지며 implementation파일의 컴파일 결과로 생성된 실행코드가 담겨있다.

3. execution 파일
	.exe, .out, .com등을 확장자로 가지며 여러개의 object파일을 통합해서 만들어지는 최종파일로 실행이 가능.

4. library 파일
	.dll(윈도우), .so(리눅스)등을 확장자로 가지며 실행코드를 가지고 있지만 독립적으로 실행할 수없음. 
	실행파일에 필요한 기능을 저장하거나 빌려주는 파일 

[2]. 컴파일러 vs 인터프리터

컴파일러는 사용 전에 실행파일을 미리 만들지만 인터프리터는 사용시에 실행파일을 만든다.

[3]. C언어의 장점, 단점
	장점 : (1). 실행파일의 크기가 작음 -->성능이 탁월함
	       (2). 하드웨어 제어가 쉬움 -->네트워크 프로그래밍,시스템 프로그래밍을 할 수있음.
	단점 : (1). 생산성이 떨어짐 --> 프로그램 개발에 시간이 많이 듬
